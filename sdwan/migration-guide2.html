3 Additional Info
3.1 Avoiding IP Conflicts
To prevent IP conflict issues, please verify if the allotted Default Gateway IP is currently not assigned to any existing PC in your office.
3.1.1 How to Check for IP Conflicts

Before assigning your new Gateway IP, run these steps to ensure the address is vacant.

3.1.2 For Windows Systems
1
Press the Windows Key, type cmd, and press Enter.
2
In the black window, type: ping [Your New Gateway IP]
✓ Reply from... : The IP is in use. Find the device and change its IP.
✗ Request timed out : The IP is likely clear.
3.1.3 For Ubuntu (Linux) Systems
1
Press Ctrl + Alt + T to open the Terminal.
2
Type: ping -c 4 [Your New Gateway IP]
✓ 64 bytes from... : The IP is active on the network.
✗ Destination Host Unreachable / 100% packet loss : The IP is vacant.
3.2 Disable Proxy Settings via Chrome

To ensure your new SDWAN connection works correctly, please follow these steps to disable any active proxy settings:

1
Open Chrome Settings: Click the three vertical dots (⋮) in the top-right corner of Google Chrome and select Settings.
2
Navigate to System: On the left-hand sidebar, click on System.
3
Open Computer Proxy Settings: Click on Open your computer's proxy settings.
4
Disable Proxy: Set Automatically detect settings to OFF and Use a proxy server to OFF.
5
Save: Ensure you click Save or Apply before closing the window.
3.3 Tracing the path to gateway

Tracing the path to your gateway (or any IP address) is a classic networking move. It's the best way to see exactly where your data is lagging or dropping off.

For Ubuntu Users: Before using traceroute, you need to install net-tools:
sudo apt update && sudo apt install net-tools -y

This will install traceroute and other network utilities on your Ubuntu system.

3.3.1 Windows Command Prompt (CMD)
tracert google.com
tracert etreasury.kerala.gov.in
Positive Results (Example for google.com): 
Tracing route to google.com [142.250.182.78]
over a maximum of 30 hops:
  1    <1 ms    <1 ms    <1 ms  192.168.1.1
  2     2 ms     2 ms     2 ms  10.5.141.1
  3     5 ms     5 ms     5 ms  172.16.2.1
  4    12 ms    12 ms    13 ms  72.14.194.1
  5    15 ms    15 ms    14 ms  142.250.182.78
Trace complete.
3.3.2 Ubuntu Terminal (After installing net-tools)
traceroute google.com
traceroute etreasury.kerala.gov.in
Positive Results (Example for google.com): 
traceroute to google.com (142.250.182.78), 30 hops max, 60 byte packets

Note: The actual IP addresses and response times will vary based on your network configuration and location.
