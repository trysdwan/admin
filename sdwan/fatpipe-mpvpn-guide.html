Overview

FatPipe® MPVPN is a high-speed router-clustering device from FatPipe Networks. It is the ultimate solution for companies that want the highest levels of WAN redundancy, reliability, and speed for data traffic directed from the network to the Internet as well as data traffic directed to servers hosted internally.

Key Capability: MPVPN aggregates any combination of DS3, T1, E3, E1, DSL, OCx, ISDN, wireless, 3G, 4G, and cable lines. It enables dynamic data transmission over multiple paths for combined speed and redundancy.

Available Throughput Options

  • Small companies and branch offices - Starting at 5 Mbps
  • Enterprise level customers - Up to 40 Gbps

Chapter 1: Initial Setup

This chapter provides you with the information required to setup the cable connections and the initial configuration for FatPipe MPVPN.

Quick Install Instructions

  1. Connect the MPVPN unit to a UPS outlet and power on
  2. Connect LAN interface to your local network and WAN interfaces to your WAN routers
  3. Configure Management PC with IP address 192.168.0.10, Subnet Mask 255.255.255.0, Gateway 192.168.0.1
  4. Point web browser to http://192.168.0.1
  5. Login with username "Administrator" (case-sensitive, no password by default)
  6. Configure WAN interfaces with IP Address, Subnet Mask, and Default Gateway
  7. Configure LAN interface (keep default 192.168.0.1 if no conflict)
Important: Please refer to the Preinstall Worksheet included in your customer packet. If you want a FatPipe Technical Support Engineer to assist with installation, fill out the Preinstall Worksheet at least 72 hours prior to installation.

Chapter 2: Interfaces

LAN Configuration

The Interface section is where you configure settings for the LAN and WAN interfaces of MPVPN.

  • Enable Proxy ARP - Default enabled, respond to ARP requests for any IPs belonging to WAN subnets
  • VLAN - Virtual Local Area Network support (Range 0 to 4096)
  • Enable DHCP Relay - Relay DHCP requests from LAN to DHCP server on WAN side
  • IPv4/IPv6 - Configure IP addresses and subnet masks

WAN Configuration

  • ISP Name - Give name to the WAN interface
  • Route Test - When to perform route test (Always / On Primary Failure)
  • Link Stabilizing Factor - Number of consecutive failures/successes before changing line status
  • Weight - For Weighted Load balancing algorithm
  • Spillover Priority Level - Assign priorities to prevent line saturation
  • Bandwidth (kbps) - Upload/Download bandwidth for QoS

Chapter 3: System

Users & Authentication

Manage user accounts with Administrator or User privileges. Supports LDAP and Active Directory Services for centralized authentication.

Security Features: Account lockout threshold, minimum password length, and mixed password requirements.

Unit Failover

Configure MPVPN units to automatically failover in case of hardware failure. At least two units required for implementation.

  • Group ID - Uniquely identifies failover group (1-255)
  • Access IP/Mask - Private subnet common to both units
  • Role - Primary or Backup
  • Stateful Failover - Ensures all sessions failover transparently

DHCP Server

Built-in DHCP Server to assign IP addresses to devices on your LAN.

Chapter 4: Load Balancing

Load Balancing Algorithms

AlgorithmDescriptionBest For
Round RobinSends sessions down lines in rotating orderSimilar speed connections
Response TimeBalances based on average response timeUnequal speed connections
Fastest RoutePer-destination balancing, fastest line for each destinationEnsuring fastest route per session
WeightedBalances traffic in proportion to defined WAN weightsControlled traffic distribution

SmartDNS™

Patented technology providing inbound load balancing to public facing servers in the LAN.

  • Load Balancing - Advertises different paths into a host on LAN
  • Speed - Speeds up delivery of inbound traffic
  • Failover - Dynamically senses failures and adjusts DNS replies

Chapter 5: Routing

Inbound Policy

Configured to allow access from WAN to servers or machines on the LAN. By default, FatPipe denies all inbound connections.

Outbound Policy

Applies to any traffic initiated on the LAN side going out. Supports QoS rules and various traffic modes.

// Example Outbound Policy parameters • Protocol: TCP/UDP/ALL • Source IP/Mask: Specify source network • Destination IP/Mask or Domain Name • Traffic Mode: Interface Priority / Interface Specific / Mixed Priority

VPN Configuration

FatPipe VPN allows site-to-site VPNs and remote user (Road Warrior) connections.

  • Supports Pre-Shared Secret and RSA Signature authentication
  • NAT-Traversal (Auto or Forced mode)
  • IPSec Certificates for user-specific authentication

MPSeC (Multi-Path Security)

Provides significantly more security of data transmission by creating multiple, independent data pathways between two or more locations.

Chapter 7: Quality of Service (QoS)

QoS is an add-on feature that allows you to prioritize WAN traffic, essential for real-time traffic including voice and video.

  • Priority Levels: 0 (highest) to 7 (Best Effort)
  • Committed Rate (CR): Guaranteed bandwidth (min 8 kbps, max 90% of link bandwidth)
  • Burst Rate: Upper limit for available bandwidth
  • Layer 7 QoS: Application-level classification for 180+ predefined applications

Chapter 8: Site Load Balancing

Optional add-on feature for inbound connectivity to Internet accessible servers. Allows Site Failover between geographically separate locations.

Site Load Balancing shares weighted traffic between two sites utilizing all lines available at each site.

Chapter 9: Server Load Balancing

Add-on feature for inbound connection load balancing. Supports HTTP and TCP-based applications.

  • Balance Methods: Round Robin, Least Connection, Source
  • Balance Modes: TCP mode, HTTP mode
  • Health Checks: SSL Check, SMTP Check, HTTP Check

Chapter 10: VPN

Add-on feature supporting site-to-site VPNs and mobile VPN clients (Road Warriors).

Supported Clients: Greenbow VPN Client, Windows XP/VPN Client, Windows Vista (Business & Enterprise) VPN Client

Chapter 14: Central Manager

Add-on module allowing management of all FatPipe appliances from a single console.

  • Organize units into groups with secret key for secure communication
  • Global Outbound Policy - Apply single policy to one or all devices
  • Global QoS - Apply QoS policies to multiple sites
  • Monitoring and alerting capabilities
